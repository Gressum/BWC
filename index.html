Mac Setup

Installer alt i rekkefølge. Hvert steg tar 2–5 minutter.

1

Installer Xcode Command Line ToolsXcode Command Line ToolsEn pakke med grunnleggende utviklerverktøy fra Apple. Inkluderer Git (versjonskontroll), kompilere (oversettere som gjør kode til programmer), og andre nødvendige verktøy.🔧 Grunnmuren du trenger før du kan bygge noe

Åpne TerminalTerminalEt program på din Mac der du skriver tekstkommandoer i stedet for å klikke. All installasjon i denne guiden gjøres her.💬 En chat med datamaskinen din (Cmd+Space → skriv "Terminal") og kjør:

Terminalxcode-select --install

Et popup-vindu dukker opp. Klikk «Install».

▶ Hvorfor trenger jeg dette?
🤔 Forklaring

Nesten alle utviklerverktøy på Mac forventer at Command Line Tools er installert. Uten dette får du feil når du prøver å installere Homebrew, Node.js, eller bruke Git. Du gjør det én gang og glemmer det.

Du trenger ikke den fulle Xcode-appen (12 GB) ennå — det er bare nødvendig i fase 5 for iOS.

2

Installer HomebrewHomebrewEn pakke-manager for Mac. Installerer utviklerverktøy med én linje kode i Terminal, i stedet for å søke og laste ned manuelt.🍺 En app-butikk for utviklerverktøy

Terminal/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Følg instruksjonene. Kan be deg kjøre to ekstra linjer etterpå — kopier og lim inn dem.

▶ Hvorfor trenger jeg dette?
🤔 Forklaring

Mac har ingen innebygd måte å installere utviklerverktøy. Homebrew fyller det gapet. Etterpå skriver du bare brew install noe for å installere Node.js, Git, GitHub CLI og hundrevis av andre verktøy. Standard for alle Mac-utviklere.

3

Installer Node.jsNode.jsEt program som kjører JavaScript utenfor nettleseren. JavaScript driver de fleste nettsider. Med Node.js bygger du hele apper, servere og verktøy.⚡ En motor som driver JavaScript-kode på maskinen din via nvmnvm — Node Version ManagerLar deg installere og veksle mellom ulike versjoner av Node.js. Ulike prosjekter kan kreve ulike versjoner.🔄 En fjernkontroll for Node-versjoner

Terminal# Installer nvm curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash source ~/.zshrc # Installer nyeste stabile Node nvm install --lts nvm use --lts # Sjekk at det fungerer node --version npm --version
▶ Hvorfor trenger jeg dette?
🤔 Forklaring

Moderne web-verktøy er bygget med JavaScript. Node.js lar deg kjøre dem på din Mac. Når du bygger med React eller Next.js, gjør Node.js arbeidet bak kulissene.

npm (Node Package Manager) følger med. Det er en nettbutikk for kode-pakker — tusenvis av ferdige komponenter du bruker i prosjektet.

--lts = «Long Term Support» — den mest stabile, trygge versjonen.

4

Installer Claude CodeClaude CodeAnthropics AI-verktøy som bor i Terminal. Skriv hva du vil bygge på vanlig norsk/engelsk, og Claude skriver koden. Kan opprette filer, installere pakker, kjøre tester — direkte i prosjektet ditt.🤖 En kompetent programmerer-assistent i Terminal

Terminal# Native installer (auto-oppdaterer) curl -fsSL https://cli.anthropic.com/install.sh | sh claude --version # Sjekk claude # Start → åpner nettleser for innlogging
⚠️ Krever betalt plan

Claude Code trenger Claude Pro ($20/mnd) eller Claude Max ($100–200/mnd). Free-planen gir ikke tilgang.

💼 Forretnings-Steg: Registrer Firma

Når du vet hva du bygger, registrer firma. Du trenger org.nr for fakturering, domene (.no), og bankkontoer.

  • 🇳🇴 Enkeltpersonforetak (ENK) — Enklest: registrer via Altinn.no → Samordnet registermelding. Koster ~2 200 kr (Enhetsregisteret). Etternavnet ditt må være i firmanavnet, men du kan ha et eget markedsføringsnavn Norge
  • 🇳🇴 Aksjeselskap (AS) — Bedre hvis dere er to (du + broren): begrenset ansvar, koster 30 000 kr i aksjekapital + ~6 000 kr i gebyrer Norge
  • 🇺🇸 LLC (USA) — Trengs for App Store i USA. Delaware/Wyoming mest populært. Koster $50–200 i statlig gebyr + ~$100/år registered agent USA
▶ ENK vs AS — hva passer for oss?
🤔 Valget

ENK passer for: én person, lite risiko, vil teste en idé raskt. Du er personlig ansvarlig for gjeld.

AS passer for: to+ personer (du og broren!), investorer later, ansvar begrenset til aksjekapital. Anbefalt om appen skal tjene penger.

Tips: Start med ENK for å teste idéen. Gjør om til AS når dere har inntekter.

▶ Hvorfor trenger jeg dette?
🤔 Forklaring

Claude Code er hjertet av hele guiden. I stedet for å lære programmering fra scratch, forteller du Claude hva du vil — og den bygger det. Den leser hele prosjektet ditt, forstår konteksten, og kan gjøre alt fra å lage en ny app til å fikse bugs.

Fungerer best med tydelige instruksjoner og et CLAUDE.md-dokument (mer i fase 3).

5

Installer VS CodeVisual Studio CodeGratis teksteditor fra Microsoft for å skrive kode. Fargekodet syntaks, fil-utforsker, innebygd Terminal, tusenvis av utvidelser. Verdens mest brukte kode-editor.📝 Microsoft Word, men for kode

Last ned fra code.visualstudio.com → dra til Applications. Installer Claude Code-extensionExtension (utvidelse)Tillegg du installerer i VS Code for ekstra funksjonalitet — fargetemaer, språkstøtte, AI-hjelp og mer.🧩 Puslespillbiter for VS Code: ⇧⌘X → søk «Claude Code» → Install.

▶ Hvorfor trenger jeg begge?
🤔 Forklaring

VS Code er der du ser og forstår koden Claude lager. Mens Claude Code i Terminal gjør arbeidet, lar VS Code deg se alle filene, gjøre manuelle endringer, og ha oversikt. Du trenger begge — Claude som bygger, VS Code som viser.

6

Sett opp GitGitVersjonskontroll-system. Holder styr på alle endringer i koden slik at du kan angre, sammenligne, og samarbeide. Hver «lagring» kalles en commit.💾 Ctrl+Z på steroider — gå tilbake til hvilken som helst versjon & GitHubGitHubNettside som lagrer Git-prosjektene dine i skyen. Andre kan se koden, og tjenester som Vercel kobler seg direkte til for å publisere appen.☁️ Sky-lagring for kode

Terminalgit config --global user.name "Ditt Navn" git config --global user.email "din@epost.no" brew install gh gh auth login # Velg: GitHub.com → HTTPS → Login with browser
▶ Hvorfor trenger jeg dette?
🤔 Forklaring

Git redder deg. Ødela Claude noe? git checkout . tar deg tilbake. Det er et sikkerhetsnett.

GitHub gjør at koden lever i skyen, ikke bare på din Mac. Mister du Macen? Koden er trygg. Vercel henter koden direkte fra GitHub for publisering.

gh (GitHub CLI) lar deg gjøre GitHub-ting fra Terminal — opprette repos, pushe kode — uten nettleseren.

💼 Forretnings-Steg: Sikre Navnet Ditt Tidlig

Før du lager kode — sikre navnet! Det hjelper ikke å bygge en app hvis noen andre eier navnet.

  • Sjekk tilgjengelighet — Søk på navnesok.no (Norge) for firma+domene+varemerke samtidig Alle
  • Registrer domene FØR firma — Folk overvåker nye firmaregistreringer og kjøper domener. Sikre .no og .com først Alle
  • .no domene — Via Domeneshop eller Uniweb. Fra ~100–300 kr/år. Krever norsk adresse eller org.nr 🇳🇴 Norge
  • .com domene — Via Namecheap eller Cloudflare. ~$10–15/år 🇺🇸 Internasjonalt
▶ Rekkefølgen er viktig!
📋 Anbefalt rekkefølge

1. Sjekk at navn er ledig (navnesok.no + domene)
2. Kjøp domene (.no og .com)
3. Registrer firma i Brønnøysund
4. Registrer varemerke hos Patentstyret

Grunnen: Noen aktører overvåker nye firmaregistreringer og kjøper matchende domener for å selge dem dyrt tilbake. Kjøp domene først!

Bygg Appen

Bruk Claude Code til å lage en fungerende web-app fra scratch.

1

Opprett prosjektmappe

Terminalmkdir -p ~/Projects/min-app cd ~/Projects/min-app
▶ Hva gjør disse kommandoene?
📖 Forklaring

mkdir -p = «make directory» — oppretter en mappe. -p lager alle undermapper som trengs.
cd = «change directory» — flytter deg inn i mappen.
~/ = din hjemmemappe (f.eks. /Users/dittnavn/).

2

Start Claude Code og beskriv appen

Terminalclaude

Du ser en > prompt. Nå snakker du vanlig språk:

Claude Code — Eksempel"Lag en Next.js web-app med Tailwind CSS. Jeg trenger: - En landingsside - Brukerinnlogging med Supabase - Et dashbord for innloggede brukere Sett opp hele prosjektet med alle nødvendige konfigurasjonsfiler."
▶ Hva skjer egentlig her?
🤔 Bak kulissene

Claude gjør automatisk:
1. Kjører npx create-next-app — setter opp prosjektstrukturen
2. Installerer pakker — Tailwind, Supabase-klient osv.
3. Oppretter filer — sider, komponenter, konfigurasjon
4. Skriver fungerende kode

Du kan iterere: «Gjør knappen blå», «Legg til en prisliste», «Fiks denne feilen».

3

Lagre og last opp til GitHub

Terminalgit init git add . git commit -m "Første versjon" gh repo create min-app --public --source=. --push
▶ Hva gjør hver linje?
📖 Linje for linje

git init — Start Git-tracking i mappen
git add . — Marker alle filer for lagring (punktum = «alt»)
git commit -m "..." — Lagre snapshot med en melding
gh repo create — Opprett GitHub-repo og last opp
--public — Alle kan se (bruk --private for hemmelig)
--push — Last opp med én gang

4

Kjør lokalt og utvikle videre

Terminalnpm run dev # Åpner på http://localhost:3000

Åpne localhost:3000localhostSpesiell nettadresse som betyr «denne datamaskinen». :3000 er portnummeret. Bare du ser den — den er ikke på internett ennå.🏠 Privat forhåndsvisning, bare for deg i nettleseren. Be Claude Code om endringer i vanlig språk.

▶ Hva skjer med «npm run dev»?
🤔 Forklaring

npm run dev starter en utviklingsserver:
1. Kompilerer koden — oversetter React/Next.js til HTML/CSS/JS
2. Serverer filer — lokal webserver på localhost:3000
3. Hot reload — endrer du en fil, oppdateres nettleseren automatisk

Stopp med Ctrl+C.

📄 Lag en CLAUDE.md-fil

Opprett i prosjektets rot. Claude Code leser den automatisk — gir mye bedre resultater:

CLAUDE.md# Min App ## Hva appen gjør En app som lar brukere [beskrivelse]. ## Tech stack - Next.js 14 med App Router - Tailwind CSS for styling - Supabase for database og auth - Stripe for betalinger - Deployet på Vercel ## Konvensjoner - TypeScript for alle nye filer - Komponenter i src/components/ - Norsk UI, engelske variabelnavn

💼 Vanlig å Inkludere i Denne Fasen

Mens du bygger, tenk på disse tingene som ofte glemmes men er viktige fra starten:

  • Brukervilkår & Personvernerklæring — Kreves av GDPR (Norge/EU) og App Store. Be Claude lage et utkast basert på appen din Påkrevd
  • Cookie-banner — Kreves i EU/EØS for alle nettsider med analytics eller tredjepartscookies Norge/EU
  • Bruker-innlogging (Auth) — Supabase Auth er gratis opptil 50 000 brukere. Vanlig å sette opp tidlig Anbefalt
  • Feilhåndtering — Tomme tilstander, lastehjul, feilmeldinger. Brukere hater blanke sider Viktig
  • Responsivt design — Test på mobil tidlig. Tailwind gjør dette enkelt med md: og lg: prefixer Viktig

💼 Forretnings-Steg: Domene, Sikkerhet & Betalinger

Nå som appen er live, er det tid for de profesjonelle detaljene:

  • 🌐 Koble eget domene — I Vercel: Settings → Domains → legg til «dinapp.no». Pek DNS-en hos domene-leverandøren til Vercel Viktig
  • 🔒 SSL/HTTPS — Vercel gir gratis SSL automatisk. Sjekk at hengelåsen vises i nettleseren Automatisk
  • 🔐 Sikkerhetssjekk — Be Claude kjøre npm audit og fikse sårbarheter. Gjør dette jevnlig Viktig
  • 💳 Betalingsløsning — Stripe er vanligst. Sett opp nå hvis appen skal tjene penger. 2.9% + 30¢ per transaksjon. Krever bankkonto og selskapsinfo Når du har brukere
  • 📊 Analytics — Vercel Analytics (gratis) eller Plausible (GDPR-vennlig, €9/mnd) — se hvem som bruker appen Anbefalt
  • 📧 Transaksjonell e-post — Resend.com (3 000/mnd gratis) for kvitteringer, velkommen-e-post, passord-reset Anbefalt
▶ Domene-registrering i detalj
🌐 Norge vs USA

🇳🇴 .no domene (Norge):
Registrer via Domeneshop.no, Uniweb.no, eller Domene.no. Pris: ~100–300 kr/år. Krav: norsk adresse eller organisasjonsnummer. Privatpersoner trenger PID fra Norid. Maks 5 .no-domener per person.

🇺🇸 .com domene (Internasjonalt):
Namecheap.com (~$10/år), Cloudflare Registrar (~$10/år, ingen påslag), eller Google Domains. Ingen krav til nasjonalitet. Anbefalt: kjøp BÅDE .no og .com for å beskytte merket.

Koble til Vercel: Logg inn hos domene-leverandøren → DNS → legg til CNAME eller A-record som Vercel oppgir.

Xcode: Product → ArchiveArchiveXcode pakker appen i en ferdig «pakke» (IPA-fil) som kan lastes opp til Apple for godkjenning.📦 Pakke appen klar for levering → Distribute App → App Store Connect. Legg til skjermbilder og beskrivelse.

⚠️ App Store-krav

Apple avviser apper som bare viser en nettside. Legg til offline-modus, push-varsler, eller native navigasjon.

💼 Forretnings-Steg: App Store-Krav & Juridisk

Apple er strenge. Ha dette klart FØR du sender inn:

  • Privacy Policy URL — Offentlig lenke til personvernerklæring. Kreves av Apple. Ha den på dinapp.no/personvern Påkrevd
  • Aldersgrense (Age Rating) — Må fylle ut spørreskjema i App Store Connect Påkrevd
  • Skjermbilder — Spesifikke størrelser for iPhone og iPad. Minst 3 per enhet Påkrevd
  • Eksportregelverk — Appen bruker HTTPS (kryptering) → svar «Ja» på spørsmål om eksportkryptering Påkrevd
  • Betalinger i appen — Apple krever at digitale kjøp bruker In-App Purchase (Apple tar 15–30%). Fysiske varer kan bruke Stripe direkte Ved salg
  • DUNS-nummer — Gratis bedrifts-ID fra Dun & Bradstreet. Kreves for Apple Developer som organisasjon (ikke person) Organisasjoner

▶ Varemerke og Patent — trenger jeg det?
🛡 Beskytt idéen din

Varemerke beskytter navnet og logoen (ikke idéen). Søk hos Patentstyret (Norge, fra 3 800 kr) eller USPTO (USA, $350/klasse). Behandlingstid: 4–8 måneder.

Patent beskytter en unik teknisk løsning. Dyrt (50 000+ kr) og tar 2–4 år. Relevant kun hvis appen har helt ny teknologi — ikke for de fleste web-apper.

Anbefaling for de fleste:
✅ Registrer varemerke (navn + logo) — rimelig og viktig
✅ Kjøp domener (.no + .com) — billig forsikring
⚠️ Patent — bare hvis du har oppfunnet noe teknisk nytt

Rekkefølge: Domene → Firma → Varemerke → (Patent om relevant)

▶ Samarbeid med broren — hvem eier hva?
🤝 Eierskap og avtaler

Når to personer bygger sammen, avklar eierskap skriftlig:

Med ENK: Kun én eier. Broren er enten ansatt eller underleverandør. Ikke ideelt for samarbeid.

Med AS: Begge kan eie aksjer (f.eks. 50/50). Anbefalt! Lag en aksjonæravtale som dekker:
• Hvem eier hvor mye
• Hva skjer om én vil slutte
• Hvem bestemmer hva
• Hvordan fordeles inntekter

Tips: Bruk en advokat for aksjonæravtalen (~5 000–15 000 kr). Det er den billigste forsikringen mot fremtidige konflikter.

📱 Claude Code på iPhone

Claude Code er tilgjengelig direkte i Claude iOS-appen (fra oktober 2025). Du kan også bruke SSHSSH — Secure ShellEn sikker forbindelse fra telefonen til Mac-en din hjemme. Du skriver kommandoer på telefonen, men de kjøres på Mac-en — som en fjernkontroll.🔗 Fjernkontroll til datamaskinen din for full kontroll.

🅰️ Enklest: Claude iOS-appen▼

Last ned «Claude» fra App Store (gratis app, krever Claude Pro-abonnement). Claude Code kjører i skyen — ingen ekstra oppsett.

  • Skriv instruksjoner på vanlig norsk/engelsk
  • Claude lager, redigerer og tester kode
  • Perfekt for hurtige endringer og planlegging
💡 Begrensning

iOS-appen jobber i skyen, ikke direkte i ditt lokale prosjekt. Best for nye prosjekter eller når du vil planlegge og generere kode du limer inn etterpå.

🅱️ For full kontroll: SSH fra telefonen▼

📱 GitHub på iPhone

Last ned «GitHub» fra App Store (gratis):

  • Bla gjennom kode og se endringer
  • Gjennomgå og merge pull requestsPull Request (PR)En forespørsel om å slå sammen endringer. Du lager en PR slik at koden kan gjennomgås før den blir del av hoved-koden.✋ «Kan noen sjekke dette før vi bruker det?»

Trykk «.» på et repo i Safari → åpner github.dev (koderedigerer i nettleseren) → rediger → commit → Vercel auto-deployer. Fiks bugs fra sofaen!

🤖 Automatiser Samarbeidet

Sett opp en gang — jobber automatisk for alltid:

🔄 Auto-deploy med Vercel (allerede satt opp!)▼

Vercel gjør dette automatisk når du kobler GitHub-repoet:
Du eller broren pusher → Vercel bygger → Live på 30 sek. Ingen ekstra konfigurasjon nødvendig. Hver branchBranch (gren)En parallell versjon av koden. Broren jobber på sin branch, du på din. Etterpå slår dere dem sammen (merge).🌿 En kopi du kan eksperimentere med trygt får sin egen preview-URL!

🧪 Auto-testing med GitHub Actions▼

Kjør tester automatisk hver gang noen pusher kode. Be Claude sette det opp:

Be Claude Code"Sett opp GitHub Actions workflow som kjører lint og tester ved push til main. Lag filen .github/workflows/test.yml"

Claude oppretter YAML-filen. Neste gang dere pusher, kjøres testene automatisk.

🔔 Varsler mellom dere▼

GitHub sender automatisk varsler (e-post + GitHub-appen) når:
• Noen lager en Pull Request
• En PR blir godkjent eller får kommentarer
• En build feiler
• Noen tagger deg med @brukernavn

Anbefalt workflow for to:
1. Jobb på egne branches (ikke direkte på main)
2. Lag PR når noe er klart
3. Den andre godkjenner fra telefonen
4. Merge → auto-deploy via Vercel
